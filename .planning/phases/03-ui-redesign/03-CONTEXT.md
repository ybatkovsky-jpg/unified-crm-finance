# Phase 3: UI-редизайн — Context

> После стабилизации ядра (фаза 2) — перестроить навигацию и оболочку под
> масштаб ERP. Цель: фиксированный левый сайдбар + верхний поднав, акцентный
> брендовый цвет, тёмная тема, плавный motion.

## Зачем эта фаза

Фаза 1-2 оставила навигацию как плоский топ-бар с 15 пунктами — он переполняется
и не масштабируется. Для ERP нужен чёткий раздел на модули (CRM / Закупки /
Финансы / Аналитика) с поднавигацией. Плюс «сексуальный и плавный» motion и
современная тема.

## Что было (baseline)

- `nav-bar.tsx`: один `<header>` со всеми 15 ссылками в ряд.
- Монохромный grayscale (OKLCH, нулевая цветность), dark-токены в `globals.css`,
  но **без переключателя** темы.
- `tw-animate-css` подключён, framer-motion / next-themes **не установлены**.
- Все страницы используют `container mx-auto`.

## Решения пользователя

- Структура: **фиксированный сайдбар + верхний поднав** (клик по разделу →
  поднав с дочерними страницами).
- Motion: **framer-motion**.
- Тема: **акцентный цвет + dark mode toggle**.
- Бренд: **«ПРО Мебель» + иконка**.

## Стек

Next 16 (route groups `(app)/`), React 19, Base UI, Tailwind v4 (OKLCH-токены),
lucide-react, framer-motion 12, next-themes 0.4.
