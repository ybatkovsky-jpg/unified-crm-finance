# 11. Модуль Финансы / управленческий учёт

## 11.1. Назначение

Модуль Финансы — переносится из finpro с расширениями. Ведёт управленческий учёт по проектам: транзакции (доходы/расходы), бюджеты (план/факт), кассовые разрывы, P&L, импорт 1С-клиент-банк, классификация операций, закрытие периодов.

В unified-системе модуль глубже интегрируется с закупками: каждый счёт от поставщика автоматически создаёт потенциальный расход (после оплаты → `Transaction` типа `expense` с `invoiceId`). Каждая оплата от клиента — `Transaction` типа `income` с привязкой к проекту.

## 11.3. User Stories

- **US-FIN-01**: Как владелец, я хочу видеть финансовый дашборд за месяц, чтобы понимать, как работает бизнес.
- **US-FIN-02**: Как бухгалтер, я хочу загрузить 1С-выписку и увидеть, какие транзакции автоматически классифицировались, а какие требуют ручной работы.
- **US-FIN-03**: Как менеджер, я хочу видеть P&L своего проекта в реальном времени, чтобы контролировать маржу.
- **US-FIN-04**: Как владелец, я хочу увидеть кассовый календарь на месяц, чтобы понимать, будут ли деньги на зарплаты и налоги.
- **US-FIN-05**: Как владелец, я хочу закрыть месяц, чтобы зафиксировать финансовый результат и не дать сотрудникам случайно править данные.

## 11.7. Связь с закупками (интеграция)

Это ключевое отличие от исходного finpro:

1. При создании `Invoice` в закупках — система не создаёт `Transaction` (это только намерение расхода).
2. При оплате счёта (`Invoice.status=paid`, через `ApprovalRequest` + отметка бухгалтера) — автоматически создаётся `Transaction` типа `expense` с:
   - `projectId = Invoice.projectId`
   - `categoryId = "materials"` или из правила
   - `counterpartyId = Invoice.supplierId`
   - `invoiceId = Invoice.id`
   - `amount = Invoice.totalAmount`
   - `date = paidAt`
   - `source = "zakuppro"` (или `"unified_procurement"`)
   - `externalId = Invoice.id` (для дедупликации)
3. При импорте 1С-выписки — если платёж удаётся сопоставить со счётом (по ИНН + сумма + дата), `Invoice` помечается `paid`, создаётся `Transaction` со ссылкой на `Invoice`.
4. В карточке `Invoice` показываются связанные `Transaction` (для случая частичной оплаты — несколько).

## 11.8. Метрики модуля

- Выручка за период (факт vs план).
- Расходы за период (по категориям).
- Маржа средняя по бизнесу.
- Доля автоматически классифицированных транзакций.
- Кассовые разрывы: количество и сумма.
- Длительность закрытия месяца (дни от конца месяца до `PeriodClose`).
- Доля проектов с маржой выше целевой.
