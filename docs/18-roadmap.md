# 18. Дорожная карта (Roadmap)

## 18.1. Обзор

Дорожная карта рассчитана на **6 месяцев (26 недель)**, разбита на **12 спринтов по 2 недели** плюс **2 недели буфера** на неожиданные сложности (20% от срока — стандарт для проектов такой сложности). Подход — гибридный: Scrum для планирования и ретроспектив, Kanban внутри спринта для разработки.

### 18.1.1. Команда

| Роль | Количество | Занятость |
|------|------------|-----------|
| Тимлид / fullstack-архитектор | 1 | 100% |
| Fullstack-разработчик (Next.js + Prisma) | 2 | 100% |
| Python/AI-разработчик | 1 | 100% |
| QA / аналитик | 1 | 50% (part-time) |
| Product Owner (со стороны заказчика) | 1 | 25% (по необходимости) |
| DevOps (внешний консультант) | 1 | 10% (по необходимости) |

### 18.1.2. Ключевые вехи (Milestones)

| Веха | Спринт | Дата (план) | Критерий готовности |
|------|--------|-------------|---------------------|
| M1 — Архитектура и модель данных | S1 | неделя 2 | Единая Prisma-схема, ADR-01 согласован, CI/CD работает |
| M2 — MVP Backend + UI каркас | S3 | неделя 6 | Запуск приложения локально, базовые CRUD, аутентификация |
| M3 — Модуль CRM + Сделки | S5 | неделя 10 | Создание контактов и сделок, Kanban воронки |
| M4 — Модуль Договоры + Проекты | S7 | неделя 14 | Генерация PDF, создание проектов из договоров |
| M5 — Модуль Закупки (перенос из zakuppro) | S9 | неделя 18 | BOM, запросы, счета, склад, AI-агент работает |
| M6 — Модуль Финансы (перенос из finpro) | S10 | неделя 20 | Транзакции, бюджеты, импорт 1С, P&L |
| M7 — Аналитика + Уведомления | S11 | неделя 22 | Дашборды, Telegram-бот, согласования |
| M8 — Production-запуск | S12 | неделя 24 | Релиз в production, обучение, параллельный режим |

| S12-12 | Go/No-go решение по запуску | Product Owner | 2 |

**Definition of Done:**
- Production работает, пользователи обучены.
- Справочники импортированы.
- UAT успешно пройден (или есть план по критичным багам).
- Параллельный режим запущен.
- Owner принимает решение о полном переходе.

## 18.14. Буфер (недели 25–26)

Используется при:
- Срыве сроков по любому из спринтов.
- Критичных багах, найденных на UAT.
- Дополнительных требованиях заказчика.

Если буфер не нужен — используется для:
- Рефакторинга технического долга.
- Дополнительных фич из backlog.
- Улучшения покрытия тестами.
- Оптимизации производительности.

## 18.15. Backlog (после 6 месяцев)

Идеи для следующей фазы (не входят в текущий scope):

1. **IP-телефония** — интеграция с АТС, автоматическое создание лидов из звонков, запись разговоров.
2. **Электронная подпись (КЭП)** — подписание договоров без печати сканов.
3. **Мультикомпания** — поддержка нескольких юрлиц с раздельным учётом.
4. **Мобильное приложение** — для менеджеров (просмотр сделок, добавление контактов).
5. **Интеграция с Bitrix24/amoCRM** — двусторонний обмен контактами и сделками.
6. **Расширенная аналитика** — BI-дашборды, кастомные отчёты, экспорт в Power BI.
7. **Склад с штрихкодированием** — печать этикеток, сканирование при приёмке/списании.
8. **Производство** — модуль ProductionTask (производственные задания, маршрутные карты).
9. **API для внешних потребителей** — публичный API с rate limiting и квотами.
10. **WebSocket для real-time collaboration** — одновременная редакция карточек.
